Variante 1: Grid mit festen Spalten

In dieser Variante wird das Grid in drei gleich große Spalten unterteilt. Jede Spalte nimmt den gleichen Anteil des verfügbaren Platzes ein. Dies ist ideal für gleichmäßig verteilte Inhalte.

Die Anzahl der Spalten ist festgelegt, unabhängig von der Größe des Containers. Dies eignet sich besonders für Layouts, bei denen die Inhalte immer in einer festen Struktur angezeigt werden sollen.

Variante 2: Grid mit automatischen Spalten

Hier passt sich die Anzahl der Spalten automatisch an die verfügbare Breite an. Jede Spalte hat eine Mindestbreite von 100px und teilt den restlichen Platz gleichmäßig auf.

Das Grid ist flexibel und reagiert dynamisch auf die Größe des Containers. Wenn der Platz nicht ausreicht, werden die Elemente in die nächste Zeile verschoben. Dies ist besonders nützlich für responsive Designs.

Variante 3: Grid mit expliziten Zeilen und Spalten

In dieser Variante werden die Spalten und Zeilen explizit definiert. Die erste Spalte ist kleiner, während die zweite doppelt so groß ist. Zeilen können ebenfalls feste Höhen haben.

Durch die explizite Definition von Zeilen und Spalten können komplexe Layouts erstellt werden. Dies ermöglicht eine präzise Kontrolle über die Größe und Position der Inhalte im Grid.

Variante 4: Grid mit Bereichsdefinitionen

Hier werden Bereiche wie "Header", "Main" und "Sidebar" definiert. Diese Bereiche können flexibel angeordnet werden, um ein übersichtliches Layout zu erstellen.

Die Bereiche werden durch Namen definiert, die in der `grid-template-areas`-Eigenschaft verwendet werden. Dies macht es einfach, Layouts zu erstellen, die logisch und leicht verständlich sind.
